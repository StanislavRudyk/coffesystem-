# ☕ Cafe System — Преміальна екосистема для управління кав'ярнею

**Cafe System** — це сучасний програмний комплекс для автоматизації роботи кав'ярні, побудований на стеку **.NET 8** та **PostgreSQL**. Проект поєднує в собі потужність серверної архітектури Web API та елегантність десктопного інтерфейсу в стилі **Bento Grid** та **Glassmorphism**.

---

## 📽 Огляд системи

Система охоплює всі аспекти роботи закладу: від прийому замовлення бариста до глибокої аналітики продажів адміністратором.

### Ключові модулі:
- **Cafe.API** (Сервер): REST API для обробки даних, аутентифікації та роботи з базою.
- **Cafe.Launcher** (Шлюз): Естетична форма входу з вибором ролі (Admin / Manager).
- **Cafe.Client.Manager** (Каса): Інтерфейс бариста для швидкого вибору товарів та оформлення чеків.
- **Cafe.Client.Admin** (Управління): Dashboard з аналітикою, управлінням персоналом та меню.

---

## 🏗 Технічна архітектура та рішення

### 🎨 Sovereign UI Engine (GDI+)
Весь графічний інтерфейс клієнтів малюється вручну за допомогою розробленого **SovereignEngine**. Це дозволило реалізувати:
- **Ефект матового скла (Glassmorphism)** з динамічними градієнтами.
- **Bento-структуру**: адаптивні блоки з ідеальним закругленням кутів (24px).
- **Плавний скролінг (SmoothGrid)**: кастомні елементи керування для миттєвого відгуку на дії користувача.

### 🔌 API & Connectivity
- **Shared Data Layer**: Використання бібліотеки `Cafe.Shared` для уніфікації моделей даних між сервером та всіма клієнтами.
- **ApiService**: Централізований сервіс на базі `HttpClient` з автоматичною десеріалізацією та обробкою помилок.
- **Role-Based Login**: Автоматичний запуск потрібного клієнта залежно від ролі користувача, отриманої від API.

### 🗄 База даних та ORM
- **Entity Framework Core**: Робота з базою даних через Code-First підхід.
- **DbInitializer**: Автоматичне наповнення бази даними (категорії, товари, юзери) при першому запуску.

## 🐳 Швидкий запуск через Docker

Проект підтримує розгортання бази даних через **Docker Compose**, що дозволяє підняти серверне оточення однією командою.

### 1. Запуск бази даних
Перейдіть в корінь проекту та виконайте:
```bash
docker-compose up -d
```
Це запустить контейнер **PostgreSQL** з наступними параметрами:
- **DB_NAME**: `cafe_db`
- **USER**: `admin`
- **PASSWORD**: `admin`
- **PORT**: `5432`

### 2. Налаштування підключення
Переконайтеся, що в файлі `Cafe.API/appsettings.json` вказано правильний рядок підключення:
```json
"DefaultConnection": "Host=localhost;Port=5432;Username=admin;Password=admin;Database=cafe_db"
```

### 3. Запуск додатку
1. Запустіть **Cafe.API** (Сервер автоматично створить таблиці та наповнить їх даними).
2. Запустіть **Cafe.Launcher** для входу в систему.

## 🛠 Технологічний стек
- **Backend**: ASP.NET Core 8.0, EF Core, PostgreSQL.
- **Frontend**: C# WinForms, GDI+, Custom UI Controls.
- **Shared**: .NET Standard 2.0.
- **DevOps**: Docker Compose.

## 📦 Структура рішення
- `Cafe.API/` — Логіка сервера та контролери.
- `Cafe.Shared/` — Моделі, DTO та ApiService.
- `Cafe.Launcher/` — Лаунчер з авторизацією.
- `Cafe.Client.Manager/` — Фронтенд для бариста.
- `Cafe.Client.Admin/` — Фронтенд для адміністратора.
- `docker-compose.yml` — Конфігурація бази даних.
