# ☕ Cafe System — Преміальна екосистема для управління кав'ярнею

Професійний програмний комплекс (курсова робота), розроблений на базі **.NET 8** та **WinForms**, з використанням сучасної архітектури Web API та дизайнерської концепції "Bento Grid" / "Glassmorphism".

## 🏗 Архітектура проекту

Проект розділений на 5 ключових модулів для забезпечення масштабованості та чистоти коду:

1.  **Cafe.API**: Центральний сервер на базі ASP.NET Core. Керує базою даних PostgreSQL, обробляє запити на аутентифікацію, замовлення та управління складом.
2.  **Cafe.Shared**: Спільна бібліотека класів (.NET Standard 2.0). Містить моделі даних (User, Product, Order) та `ApiService` — клієнт для взаємодії з сервером.
3.  **Cafe.Launcher**: Стартовий шлюз системи з преміальною анімацією. Перевіряє права доступу та запускає відповідний клієнт (Admin або Manager).
4.  **Cafe.Client.Manager**: Робоче місце бариста. Оптимізовано для швидкого прийому замовлень за допомогою візуального каталогу та розумного кошика.
5.  **Cafe.Client.Admin**: Панель управління для власника. Містить інтерактивні графіки продажів, історію чеків та редактор меню.

## ✨ Основні можливості

### 🎨 Дизайн та UX
- **Sovereign UI Engine**: Власна графічна система для малювання закруглених елементів, градієнтів та ефекту матового скла.
- **Bento Grid**: Структурований інтерфейс, де кожен блок має своє чітке місце.
- **Анімації та Micro-interactions**: Плавні переходи та реакції інтерфейсу на дії користувача.

### 🛡 Безпека та Ролі
- **Авторизація**: Система входу з перевіркою паролів.
- **Ролі користувачів**: 
    - `Admin`: Повний доступ до аналітики та налаштувань.
    - `Manager`: Доступ до касових операцій.

### 📊 Аналітика та Управління
- **Dashboard**: Візуалізація виручки та кількості замовлень через `ChartService`.
- **Inventory Management**: Повний цикл CRUD для товарів та категорій.
- **Personnel Control**: Управління співробітниками з фільтрацією за ролями.

## 🚀 Як запустити

### 1. Підготовка бази даних
Система використовує **PostgreSQL**. Переконайтеся, що рядок підключення в файлі `Cafe.API/appsettings.json` відповідає вашим налаштуванням. База даних та початкові дані (юзери, категорії) створюються автоматично при першому запуску сервера.

### 2. Запуск Сервера
Першим ділом запустіть проект **Cafe.API**. Він повинен працювати на фоні, щоб клієнти могли підключитися.

### 3. Запуск Лаунчера
Запустіть **Cafe.Launcher**. Це основна точка входу для користувача.

### 🔑 Тестові дані для входу:
- **Адміністратор**: `admin` / `admin123`
- **Менеджер (Бариста)**: `user` / `user123`
